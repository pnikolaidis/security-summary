Today is June first, 2026, and we have five stories that security teams should be watching closely — from a critical WordPress plugin under active attack, to a major botnet takedown, and a significant surveillance policy shift out of Russia.

Our first story concerns the WP Maps Pro WordPress plugin. Security researchers flagged a critical authentication bypass this weekend, and threat actors moved fast: exploitation is already actively underway. The plugin lets site owners embed Google Maps and advanced location features on their WordPress sites, and it has sold over fifteen thousand copies on the Envato Market. The flaw allows anyone on the internet — with zero credentials — to create a full administrator account on a vulnerable site. Once an attacker has an admin account, full site compromise follows quickly: they can install malware, redirect visitors to phishing pages, or exfiltrate stored data. If your organization runs WordPress sites, this is a same-day patch priority. Update WP Maps Pro to the latest version, or deactivate the plugin until you can.

Next, Palo Alto Networks PAN-OS firewall customers are facing an active threat. CVE-2026-0257 is an authentication bypass in PAN-OS — the operating system powering Palo Alto's widely deployed next-generation firewalls. Hackers began exploiting it just four days after public disclosure, and reporting confirms that exploitation has continued for weeks. An authentication bypass in a firewall management plane is a high-severity scenario: it can give attackers full control over the devices that are supposed to be protecting your network. If you run PAN-OS, verify the patch has been applied and pull management-plane logs going back at least several weeks to look for unauthorized access.

Third, a supply chain attack is targeting developers who use OpenAI Codex. Researchers uncovered a malicious package on the npm registry called codexui-android, which presents itself as a remote web interface for OpenAI Codex. The package had accumulated over twenty-nine thousand weekly downloads before detection, giving it significant reach. Its real purpose is stealing authentication tokens from developers' machines. With those tokens, attackers gain access to the victim's OpenAI API account — including proprietary source code submitted for AI-assisted completion, stored API credits, and the ability to run up charges. As of reporting, the package remained available for download. Developers should audit their npm dependencies and rotate any OpenAI credentials that may have been exposed.

Story four is good news from the Netherlands, where law enforcement scored a major win. Dutch police, working with the National Cyber Security Center, dismantled a botnet that had infected at least seventeen million devices worldwide — computers, smartphones, tablets, and internet-of-things gear. More than two hundred command-and-control servers were seized. A botnet of that size represents serious criminal infrastructure capable of launching large-scale denial-of-service attacks, credential stuffing operations, and spam campaigns. The identities of the operators have not yet been publicly released. Organizations should check endpoint and network telemetry for any past botnet-related traffic that might indicate devices were previously part of the network.

And finally, a significant development in Russian surveillance infrastructure. Russia has greatly expanded its SORM requirements — the System for Operative Investigative Activities — Russia's legally mandated interception framework that requires telecom operators and internet service providers to install hardware giving the FSB direct access to communications traffic. The new expansion broadens both the categories of services covered and the data retention periods mandated. For organizations with employees, offices, or cloud infrastructure in Russia, this is a clear signal to ensure sensitive communications are protected with end-to-end encryption, and to review travel and remote-access policies to account for the expanded surveillance environment. Anyone relying on Russian-hosted services for any part of their security operations should treat that traffic as fully monitored.

Five stories: a WordPress plugin under active exploitation, a Palo Alto firewall vulnerability that attackers jumped on in days, a supply chain attack harvesting OpenAI developer credentials, a seventeen-million-device botnet dismantled by Dutch law enforcement, and Russia significantly widening its national surveillance network. Stay safe out there.
