Good morning. Here is your security digest for May 22nd, 2026. Today's top stories cover a lot of ground: actively exploited Microsoft Defender zero-days that need immediate patching, a coordinated Chinese state-sponsored campaign quietly compromising telecom operators with a shared Linux backdoor, a supply-chain attack that traced back to a rogue VS Code extension and ultimately reached GitHub's own internal repositories, a nine-year-old Linux kernel flaw now public that grants root access on major distributions, and the arrest of the alleged operator of one of the fastest-spreading botnets in recent memory.

Microsoft has disclosed two zero-day vulnerabilities in Microsoft Defender that are being actively exploited in the wild right now. The primary tracked identifier is CVE-2026-41091. Notably, attackers were exploiting these flaws before Microsoft's public advisory went out — meaning defenders were likely encountering them without knowing what they were up against. Microsoft has released patches and is urging immediate application. Because Defender is embedded across virtually every Windows enterprise deployment, the attack surface is enormous. Organizations should apply the latest updates without delay, audit endpoint telemetry for signs of tampering with the Defender process, and review recent endpoint alerts for activity that may have been related to this exploitation campaign. This is the highest-priority patch from today's digest.

Multiple Chinese state-sponsored threat groups have been caught deploying a new Linux backdoor called Showboat against telecom operators across the Middle East and Central Asia. What makes this campaign especially alarming is that distinct APT groups appear to be sharing the same tooling — an unusual level of coordination that suggests state-level orchestration or deliberate resource sharing between otherwise separate units. Once inside a carrier's infrastructure, Showboat installs a SOCKS5 proxy that lets attackers blend their covert traffic with legitimate network flows, making detection significantly harder. For organizations in telecom and critical infrastructure, the recommended actions are to audit Linux server baselines for unexpected processes or listeners, look specifically for SOCKS5 proxy services that shouldn't be there, and review outbound network traffic patterns for anomalous egress behavior.

A sophisticated supply-chain attack unfolded this week that started in the VS Code extension marketplace and ended with GitHub's own internal repositories being breached. The attack began with a malicious version of the Nx Console extension distributed through the VS Code marketplace. Developers who installed it had their GitHub credentials silently exfiltrated. Those stolen tokens were then used to access GitHub's internal repositories, and the attackers injected malicious code into the TanStack npm package — a widely used JavaScript library with millions of weekly downloads. GitHub has since linked the two incidents together. If you or your developers use VS Code, audit all installed extensions immediately, rotate any GitHub tokens that may have been exposed, and verify the integrity of TanStack packages in your dependency trees. Supply-chain attacks of this sophistication, starting with the developer toolchain, are becoming a recurring pattern.

Researchers have publicly disclosed a nine-year-old vulnerability in the Linux kernel tracked as CVE-2026-46333 that allows any local user to execute arbitrary commands as root. The flaw was introduced in 2017 and affects virtually every major Linux distribution in use today — enterprise servers, cloud infrastructure, containers, and developer workstations alike. Local exploitation grants full system control, enabling data theft, persistent backdoor installation, or escalation into broader infrastructure. Kernel patches are available upstream and are rolling out through distribution channels now. Prioritize kernel updates across your Linux estate, and give extra attention to shared-access systems or any environment where multi-tenant or container workloads run, since the local-access requirement is easily satisfied in those contexts.

On the law enforcement front, Canadian authorities arrested a 23-year-old Ottawa resident who goes by Dort online, alleging he built and operated the Kimwolf botnet. Kimwolf was one of the fastest-spreading botnets observed in recent years, and it was used to facilitate a range of cyberattacks. The case is a joint U.S. and Canada operation, and the investigation into the full scope of Kimwolf's activity is still ongoing. While the arrest is a meaningful disruption, botnet infrastructure often outlasts individual operators. Security teams should continue watching for Kimwolf indicators of compromise as the investigation produces more attribution details and infrastructure disclosures.

That's your digest for May 22nd, 2026. Patch Defender, update Linux kernels, rotate any exposed GitHub tokens, audit your VS Code extensions, and keep an eye on Kimwolf indicators as that investigation develops. Stay safe out there.
