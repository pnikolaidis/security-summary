This is Adam Insight with Tuesday's security digest. Five stories today — two active exploitation alerts, a patching policy shift, an operational Windows hazard, and a privacy ruling that security teams should keep an eye on.

Story one: CISA is ordering federal agencies to patch a Drupal vulnerability by Wednesday, and if you run Drupal, you should treat that deadline as your own.

The vulnerability is a SQL injection flaw in Drupal, the content management system used across government portals, universities, and enterprise web infrastructure. CISA has flagged it as actively exploited and added it to the Known Exploited Vulnerabilities catalog. The flaw is unauthenticated — no login required — and allows attackers to read arbitrary data from the database. That means credentials, session tokens, user records, and anything else the application stores. Federal agencies have until Wednesday evening. For everyone else, CISA KEV listings are a reliable signal that exploitation is real, not theoretical. Unpatched public-facing Drupal instances should be prioritized immediately.

Story two: A zero-day in a Japanese learning management system was used to plant web shells and Cobalt Strike implants, and the root cause is a lesson that applies far beyond this one product.

The vulnerability, tracked as CVE-2026-5426, was in the KnowledgeDeliver LMS. Attackers exploited it before a patch was available. The underlying problem is hard-coded ASP.NET machine keys — essentially, the application shipped with a fixed cryptographic secret baked into its configuration. Anyone who knows that key, which may have been discovered through source code analysis or prior leaks, can forge a viewstate payload that tricks the server into deserializing and executing arbitrary code. In this campaign, attackers used that code execution to install the Godzilla web shell for persistent access, then dropped Cobalt Strike Beacon for command and control. A patch is now available. But the broader warning is for any organization running ASP.NET applications: audit your machine key configuration. If you are using default keys, shared keys, or keys that have ever appeared in public documentation, you have the same exposure as the organizations that got hit here.

Story three: India's computer emergency response team is now requiring organizations to patch critical internet-facing vulnerabilities within twelve hours, and the reason they're setting that timeline is directly tied to artificial intelligence.

CERT-In, India's national cybersecurity authority, published new guidelines mandating that when a critical vulnerability is flagged in internet-exposed systems, organizations should patch it within twelve hours where feasible. The guidance explicitly names AI tools and large language models as the driving factor: threat actors are using them to automate vulnerability scanning, proof-of-concept development, and exploitation at a pace that makes traditional patch windows — days or weeks — dangerously slow. This is a significant shift. The prevailing standard has been a thirty-day remediation window for critical flaws, or up to fifteen days for federally mandated patches in the United States. Twelve hours is a different order of magnitude. Security teams should start thinking now about what it would take to respond that quickly for their highest-exposure attack surface.

Story four: If you are running Windows Server 2016 and have not yet applied the May 2026 security update, hold off. If you already have, watch your domain controllers.

Microsoft has confirmed that the May 2026 security update, identified as KB5087537, can cause domain controller lookup failures on Windows Server 2016 systems after installation. In an Active Directory environment, domain controller failures cascade quickly into authentication problems across every service that relies on Kerberos or Windows authentication. Users cannot log in, applications fail to authenticate to databases and APIs, and anything that depends on group policy breaks. Microsoft has acknowledged the issue and is tracking it. Organizations should check the Windows release health dashboard for the current guidance before deploying this update in Server 2016 environments.

Story five: California is amending its age-verification law to exempt Linux after the open-source community pushed back against a requirement that would have forced operating systems to collect user ages.

The original legislation would have required operating systems to implement age verification at the OS level. That phrasing did not carve out exceptions for open-source software, meaning Linux distributions — which are maintained by communities rather than corporations — would have been technically required to implement something that is architecturally incompatible with how open-source software is built and distributed. The response from the Linux community was swift and well-organized, and the lawmaker who wrote the original bill has now proposed an amendment to exempt Linux. For security and privacy professionals, the significance goes beyond Linux. This is the first serious legislative attempt in the United States to push age-verification enforcement down to the operating system layer. How that boundary is drawn — whether at the app, the platform, or the OS — has direct implications for privacy architecture, secure-by-design principles, and how future regulation gets written.

That's your Tuesday digest. Patch Drupal, audit your ASP.NET machine keys, note the twelve-hour patching signal from India, hold the Windows Server 2016 update, and keep an eye on the California legislation. Stay safe out there.
