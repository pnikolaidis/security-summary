Good morning. Ada Iverson here. Wednesday's signal, coming up. Here is your security digest for May twenty-seventh, twenty twenty-six. Today is a heavy patch day — a wave of serious vulnerability disclosures is hitting network security software, enterprise middleware, and widely-deployed web platforms, with several bugs already confirmed as actively exploited or carrying the highest possible severity scores.

Story one: FastNetMon Community Edition, the open-source DDoS detection platform used by internet service providers and network operators worldwide, has received eight new CVEs — including two critical memory corruption flaws and an API that lets any attacker on the network take full control with no credentials required.

FastNetMon is the software that sits at the edge of a network and detects denial-of-service attacks in real time, automatically triggering router configurations to block malicious traffic. The two most severe bugs — both scored 9.8 out of ten — are memory corruption issues inside the BGP packet decoder. BGP is the routing protocol that holds the internet together, and FastNetMon reads raw BGP messages to understand network topology. A specially crafted message can trigger a stack overflow or a heap overflow, potentially allowing an attacker to execute arbitrary code on the monitoring host. On top of that, the software's management API runs on port 50052 with no authentication whatsoever. The source code actually contains a comment that says, and I quote, "listen without authentication." Anyone on the same network segment can connect to that port and issue commands. There are also command injection bugs in the router integration plugins for Mikrotik and Juniper, meaning an attacker could hijack router configurations during what looks like a normal DDoS mitigation response. FastNetMon has not yet released a patch. Why it matters: organizations running FastNetMon for DDoS protection may unknowingly be exposing an unauthenticated remote control interface to their network. Firewall port 50052 and restrict who can send BGP traffic to the monitor until a fix is available.

Story two: IBM WebSphere and HTTP Server have received emergency patches, including a critical remote code execution flaw scored 9.8 that requires no credentials and no special preconditions to exploit remotely.

IBM released fixes for nine CVEs affecting IBM HTTP Server versions 8.5 and 9.0 and the WebSphere Liberty web server plugins. The headline vulnerability is an unauthenticated remote code execution bug in the WebSphere Liberty plugin — an attacker anywhere on the internet who can reach the server can potentially run arbitrary code. A separate bug also enables both remote code execution and denial of service in IBM HTTP Server itself. There are five additional vulnerabilities covering buffer overflows, memory corruption, and denial of service in the same product line. In a separate advisory, IBM Engineering Lifecycle Management — the product suite organizations use for software requirements, development tracking, and compliance — also received a critical authentication bypass scored 9.8 that lets unauthenticated users access protected resources. Why it matters: IBM WebSphere and HTTP Server power a significant share of enterprise Java infrastructure around the world. If your organization runs on-premises IBM middleware, this patch set should be at the very top of your maintenance queue this week.

Story three: CISA has given U.S. federal agencies just four days to patch an actively-exploited vulnerability in the LiteSpeed plugin for cPanel.

The Cybersecurity and Infrastructure Security Agency added the flaw to its Known Exploited Vulnerabilities catalog and issued a binding directive requiring all federal civilian agencies to remediate by May thirty-first. The vulnerability affects the LiteSpeed user-end plugin, an extension for cPanel — the web hosting control panel that millions of websites run on. Attackers are actively exploiting this flaw right now in real-world attacks. cPanel gives the server operator control over websites, databases, email accounts, and SSH keys, so a successful compromise of a cPanel server is a broad breach, not just a single-website incident. While the directive formally applies only to federal agencies, any organization running cPanel with the LiteSpeed plugin enabled should treat this as critical-priority. Apply the vendor's patch or disable the plugin immediately. Why it matters: actively-exploited control panel flaws mean attackers are already in the wild scanning and compromising vulnerable servers. Delays here have real consequences.

Story four: Joomla has patched three critical privilege-escalation vulnerabilities — all scored 9.8 — that allow any authenticated user, even a basic subscriber, to promote themselves to full administrator.

Joomla is one of the world's most widely used content management systems, powering millions of websites. All three bugs live in the same component — a batch processing and group management endpoint in the user management module. The vulnerability is an access-control failure: the endpoint doesn't properly check whether the user calling it has the authority to modify user roles. An attacker who has any valid login — even a low-privilege reader account — can call this endpoint and assign themselves administrator-level access. From there, they can install plugins, modify content, export the database, or use Joomla's server-side capabilities to move deeper into the hosting environment. Joomla has released a patched version. Why it matters: three separate CVEs pointing at the same component means the attack surface is wide and the fix is well-defined. Update Joomla immediately.

Story five: A critical vulnerability in KubeVirt — scored 9.9, the second-highest possible — allows an attacker to escape their Kubernetes namespace and gain read and write access to the underlying host server.

KubeVirt is the extension that lets Kubernetes clusters run traditional virtual machines alongside containers. It's widely used in Red Hat OpenShift environments and large enterprise Kubernetes deployments. The vulnerability is in the component that manages console connections to those virtual machines. When a user connects to a VM console, the handler doesn't properly validate symlinks at the socket path. An attacker with only namespace-level edit permissions — the kind of access a developer typically has in a shared cluster — can replace the expected socket file with a symlink pointing to any file on the physical host. Once the handler follows that symlink, the attacker gains read and write access to that file. That includes node certificates, service account tokens, and Kubernetes secrets stored on disk, all of which could be used to compromise the entire cluster's control plane. A patch is available. Why it matters: Kubernetes namespace boundaries are the fundamental security isolation model for multi-tenant cloud infrastructure. This bug defeats that model with developer-level access.
