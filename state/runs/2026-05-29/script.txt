Good morning. Here is your security digest for May 29, 2026. Today's theme is cascading trust failures: a major enterprise vendor just dropped a wall of critical patches, two popular open-source tools have serious unresolved vulnerabilities, and attackers are abusing the very management infrastructure meant to secure organizations.

Oracle just released a massive Critical Patch Update covering more than twenty high- and critical-severity vulnerabilities across its E-Business Suite, REST Data Services, and Database Server products. The most alarming is a CVSS 10.0 flaw in Oracle REST Data Services. That means it is the worst possible severity score — and it allows an unauthenticated attacker to fully take over the REST Data Services server with no credentials at all, with the damage spilling across into connected Oracle infrastructure. Several other flaws in the same product family rate 9.9, also requiring only low-privilege network access. Oracle Hospitality's OPERA 5 property management system, which many hotels rely on, has a CVSS 9.8 unauthenticated HTTP takeover. If your organization runs Oracle E-Business Suite on version 12.2 or Oracle REST Data Services anywhere between 24.2 and 26.1, patching this is not optional.

Our second story involves a very public fight over who controls the clock on vulnerability disclosure. A security researcher published unpatched zero-day vulnerabilities in Windows — and then had their GitHub account removed, apparently in response. Microsoft has come out publicly in support of Coordinated Vulnerability Disclosure, which means giving vendors time to fix bugs before going public. Critics fired back hard. A widely shared essay called Microsoft's stance on zero days a, quote, dumpster fire of their own making, pointing to a pattern of missed patch deadlines, dismissive responses to researchers, and now, using platform ownership to suppress disclosures. The story exploded on Hacker News with over four hundred points and nearly two hundred comments. GitHub is owned by Microsoft, which makes the account removal feel less like a content moderation decision and more like a leverage move. The vulnerabilities themselves — Windows zero-days — remain unpatched and undisclosed in full.

Third today: an unpatched zero-day in Gogs, a popular self-hosted Git service used by development teams who want a lightweight alternative to GitLab or Gitea. Researchers at Rapid7 found that any authenticated user — not just an admin, any authenticated user — can achieve remote code execution on the server. The flaw rates 9.4 on the CVSS scale. There is no patch. There is no CVE yet. This is a true zero-day. Gogs instances that are exposed to the internet or to untrusted users are at real risk right now. The recommended mitigation is to restrict who can authenticate until a fix is available.

Fourth: attackers are actively exploiting a patched authentication bypass in FortiClient Enterprise Management Server, tracked as CVE-2026-35616, to deploy a previously unknown credential stealer called EKZ. What makes this particularly nasty is the delivery mechanism. Fortinet's EMS is the system that pushes software and policies to all the managed endpoints in a network. By compromising EMS, attackers can deliver malware to every managed device in one shot — the exact opposite of what the system is supposed to do. The malware is disguised as a legitimate Fortinet binary to avoid suspicion. If you're running FortiClient EMS and haven't applied the patch for this CVE, consider the entire managed fleet potentially exposed.

And rounding out today's digest: a supply chain double-tap hitting both the dot-NET and JavaScript package ecosystems. A malicious package on NuGet is impersonating the official C-sharp SDK for Sicoob, one of Brazil's largest banking networks. Versions 2.0.0 through 2.0.4 of Sicoob dot S-D-K quietly exfiltrate P-F-X certificates and client IDs — credentials used for financial authentication — to an attacker-controlled server. Separately, malicious npm packages have been found targeting cloud credentials from developer environments. Both campaigns hit at the same time, which suggests either coordination or copycat behavior. If your team works in either ecosystem and recently added a new package touching banking integrations or cloud APIs, it's worth auditing those additions now.

That's your digest. Stay safe out there.
